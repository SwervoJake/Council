## Pattern: Three-Scenario Before Any Initiative
- **Confidence:** 0.5
- **Applications:** 0
- **Last updated:** 2026-03-04
- **Pattern:** Before approving any initiative with meaningful cost, model best/base/worst scenarios with explicit probabilities. Never proceed on the base case alone.
- **Conditions:** Any initiative with >$5K cost or >2 weeks of engineering time.
- **Counter-evidence:** Trivial, fully reversible experiments may not warrant full modeling — judgment call.
- **Source:** Factory Council architecture specification

## Pattern: Burn Multiple as Health Indicator
- **Confidence:** 0.5
- **Applications:** 0
- **Last updated:** 2026-03-04
- **Pattern:** Track Burn Multiple (net burn ÷ net new ARR) as the primary efficiency signal. <1.0 = excellent. 1.0–1.5 = good. 1.5–2.0 = concerning. >2.0 = requires immediate intervention.
- **Conditions:** Post-revenue stage. Pre-revenue companies substitute pipeline multiple.
- **Counter-evidence:** Early stage with low ARR base — absolute burn matters more than multiple.
- **Source:** Factory Council architecture specification

## Pattern: Kill Criteria Set Before Spend Begins
- **Confidence:** 0.5
- **Applications:** 0
- **Last updated:** 2026-03-04
- **Pattern:** Every investment decision includes measurable kill criteria defined BEFORE spend begins. Criteria set while objective — not while hoping a bet will recover.
- **Conditions:** Any initiative with committed spend or engineering resources.
- **Counter-evidence:** Exploration phases where the point IS to discover the right metrics.
- **Source:** Factory Council architecture specification
