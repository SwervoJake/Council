## Pattern: Traffic Light Before Proceeding
- **Confidence:** 0.5
- **Applications:** 0
- **Last updated:** 2026-03-04
- **Pattern:** Every initiative gets an explicit RED/AMBER/GREEN classification before proceeding. Unrated risk is treated as AMBER by default. The act of classification forces explicit risk acknowledgment.
- **Conditions:** Any initiative with legal, regulatory, or liability implications.
- **Counter-evidence:** Fully internal, zero-external-facing decisions with no IP or privacy implications.
- **Source:** Factory Council architecture specification

## Pattern: Distinguish Awareness from Action
- **Confidence:** 0.5
- **Applications:** 0
- **Last updated:** 2026-03-04
- **Pattern:** Always label legal flags as AWARENESS (know this, no action needed) vs. MITIGATE (specific action required) vs. ESCALATE: EXTERNAL (licensed counsel required). The distinction prevents over-lawyering low-stakes situations and under-lawyering high-stakes ones.
- **Conditions:** Every Ward output.
- **Counter-evidence:** Not applicable — this is a universal protocol, not a conditional strategy.
- **Source:** Factory Council architecture specification

## Pattern: Privacy First for Data Products
- **Confidence:** 0.5
- **Applications:** 0
- **Last updated:** 2026-03-04
- **Pattern:** Any product or feature involving user data, PII, or health data gets a privacy impact assessment before technical design is finalized. Retrofitting privacy is 3–10× more expensive than designing it in.
- **Conditions:** Any initiative touching user data collection, storage, or processing.
- **Counter-evidence:** Internal tools with no user PII — reduced assessment warranted.
- **Source:** Factory Council architecture specification
